## Incident Response

### DLP Incident: Suspected Data Exfiltration

A live DLP incident required active investigation of potential sensitive data exfiltration by staff.

Actions taken:
- Created detection rules in Teramind for unauthorised use of anonymizing browsers
- Monitored flagged users for indicators of sensitive data exfiltration activity
- Produced documented accountability evidence for management review and audit use

This was not a simulated exercise. It involved real staff, real data risk, and output that was reviewed at management level.

## Critical Incidents Resolved

**Fleet-wide internet outage (Bitdefender)**
Staff across the organisation lost internet access simultaneously with no obvious cause. Diagnosed the issue to a Bitdefender configuration change and resolved it, restoring connectivity fleet-wide.

**Access control database wipe**
The physical access control database was wiped due to bad overwrite data. Diagnosed the root cause and recovered the system.

**Uncontrolled Teramind user account spawning**
New user accounts were spawning uncontrollably due to a fault in the onboarding process. Diagnosed the cause in the Teramind configuration and resolved it.
